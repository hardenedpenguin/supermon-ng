# Debian package (supermon-ng)

Internal `.deb` packaging for ASL3+ nodes. Apache is **not** configured automatically; use your existing vhost or the template under the install tree.

## Build dependencies

On the build host:

```bash
sudo apt-get install -y debhelper dh-systemd composer npm nodejs \
  php-cli php-xml php-mbstring php-curl php-sqlite3
```

## Build

From the repository root:

```bash
./scripts/build-deb.sh
```

Output: `../supermon-ng_<version>_all.deb` (one directory above the repo).

Optional:

```bash
NODE_STATUS_INTERVAL_MINUTES=10 ./scripts/build-deb.sh
```

## Install

```bash
sudo dpkg -i ../supermon-ng_*_all.deb
sudo apt-get install -f   # if dependencies are missing
```

Install location: `/var/www/html/supermon-ng` (matches tarball installs and sudoers paths).

## Configure

1. Copy or edit `.env` (created from `.env.example` on first install).
2. Set `APP_BASE_PATH` (`/supermon-ng` for subdirectory, `/` for dedicated vhost).
3. Run path helper (also runs from `postinst` when `.env` exists):

   ```bash
   sudo /var/www/html/supermon-ng/scripts/configure-app-base-path.sh /var/www/html/supermon-ng
   ```

4. Configure Apache/nginx to serve `public/` (see `apache-config-template.conf` generated by `scripts/generate-apache-template.sh`).
5. Complete setup via the web UI or existing `user_files` configs.

## Services

```bash
sudo systemctl status supermon-ng-backend supermon-ng-websocket
sudo systemctl status supermon-ng-node-status.timer supermon-ng-database-update.timer
```

## Upgrade

```bash
sudo dpkg -i ../supermon-ng_<new>_all.deb
```

User configs listed in `debian/supermon-ng.conffiles` are preserved. Application files are replaced.

## Tarball vs .deb

| | Tarball + `update.sh` | `.deb` |
|--|----------------------|--------|
| Apache setup | Optional in `install.sh` | Manual |
| Config backup | `update.sh` | `dpkg`/conffiles |
| Composer vendor | On target | Bundled at build time |

Both can coexist on the same path; do not mix upgrade methods without backing up `user_files/`.
